.ino → .rs — translating Arduino C to Rust

In Arduino C++ you fill in setup() and loop(). In Rust you write a single main that never returns: the code before the loop is your setup, and a bare loop {} is your loop. The real shift is ownership — you take the chip's peripherals once, and the compiler guarantees nothing else touches them, so a whole class of Arduino bugs simply can't compile.

1 · setup() / loop() → fn main() -> !

One function does both jobs. -> ! means "never returns" — there's no operating system to return to.

Arduino (C++)
void setup() {
  // runs once
}

void loop() {
  // runs forever
}
Rust
#![no_std]              // no operating system / std library
#![no_main]             // we provide the entry point
use panic_halt as _;    // on panic: just stop

#[arduino_hal::entry]
fn main() -> ! {
    let dp = arduino_hal::Peripherals::take().unwrap();
    let pins = arduino_hal::pins!(dp);

    // ---- setup: runs once ----

    loop {
        // ---- loop: runs forever ----
    }
}

Peripherals::take() hands you the chip's hardware once (it returns an Option, so a second take is None). pins! splits that into named pins you move into the parts of your program that own them.

2 · pinMode(OUTPUT) + digitalWrite

A pin's mode is its type. You can't write to a pin you didn't turn into an output — it won't compile.

Arduino (C++)
pinMode(13, OUTPUT);
digitalWrite(13, HIGH);
digitalWrite(13, LOW);
Rust
let mut led = pins.d13.into_output();
led.set_high();
led.set_low();
led.toggle();

3 · delay()

Arduino (C++)
delay(1000);        // milliseconds
delayMicroseconds(50);
Rust
arduino_hal::delay_ms(1000);
arduino_hal::delay_us(50);

4 · pinMode(INPUT_PULLUP) + digitalRead

Arduino (C++)
pinMode(2, INPUT_PULLUP);
if (digitalRead(2) == LOW) {
  // button pressed
}
Rust
let button = pins.d2.into_pull_up_input();
if button.is_low() {
    // button pressed
}
// pins.d2.into_floating_input() for a plain input

5 · Serial.begin / Serial.print

ufmt is a tiny, heap-free formatter for microcontrollers. uwriteln! is println; uwrite! is print. unwrap_infallible() just says "writing to the UART can't fail".

Arduino (C++)
Serial.begin(57600);
Serial.println("Hello!");
Serial.print(value);
Rust
use arduino_hal::prelude::*;

let mut serial = arduino_hal::default_serial!(dp, pins, 57600);
ufmt::uwriteln!(&mut serial, "Hello!").unwrap_infallible();
ufmt::uwrite!(&mut serial, "{}", value).unwrap_infallible();

6 · Serial.read

You can type into CrabDuino's Output panel while a sketch is running — it arrives here over serial.

Arduino (C++)
if (Serial.available()) {
  char c = Serial.read();
}
Rust
// nb::block! waits until a byte is available
let b = nb::block!(serial.read()).unwrap_infallible();

7 · analogRead

Arduino (C++)
int v = analogRead(A0);   // 0–1023
Rust
let mut adc = arduino_hal::Adc::new(dp.ADC, Default::default());
let a0 = pins.a0.into_analog_input(&mut adc);
let v = a0.analog_read(&mut adc);   // 0–1023

8 · analogWrite (PWM)

PWM lives in simple_pwm: pick a timer, turn an output pin into a PWM pin, then set the duty 0–255.

Arduino (C++)
analogWrite(5, 128);   // 0–255 duty
Rust
use arduino_hal::simple_pwm::*;

let timer0 = Timer0Pwm::new(dp.TC0, Prescaler::Prescale64);
let mut led = pins.d5.into_output().into_pwm(&timer0);
led.enable();
led.set_duty(128);   // 0–255
